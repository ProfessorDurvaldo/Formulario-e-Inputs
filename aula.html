01

Tipos de input

Todo formulário começa com a tag <form>. Dentro dela usamos inputs, labels, selects e botões. O atributo type define o comportamento de cada campo.

📝 text

Campo de texto genérico. É o mais usado.
Use para nomes, endereços, qualquer texto livre.

<input type="text">
✉️ email

Valida automaticamente o formato de e-mail. No celular abre teclado com "@".

<input type="email">
🔒 password

Esconde o que o usuário digita. Nunca use type="text" para senhas!

<input type="password">
📞 tel

Abre teclado numérico no celular. Não valida o formato — use pattern para isso.

<input type="tel">
🔢 number

Aceita apenas números. Suporta min, max e step.

<input type="number" min="0" max="100">
📅 date

Abre um calendário nativo do navegador. Retorna data no formato YYYY-MM-DD.

<input type="date">
🎚️ range

Slider visual. Usado quando o valor exato importa menos que a faixa. Use com min, max e step.

<input type="range" min="0" max="100">
🔘 radio

Só uma opção pode ser selecionada. Todos do mesmo grupo precisam do mesmo name.

<input type="radio" name="grupo">
☑️ checkbox

Permite múltiplas seleções. Diferente do radio, não precisam ter o mesmo name.

<input type="checkbox">
📋 select

Lista de opções para o usuário escolher. Use <option> dentro do <select>.

<select><option>...</option></select>
📄 textarea

Campo de texto com múltiplas linhas. Diferente do input, possui tag de fechamento.

<textarea rows="4"></textarea>
📁 file

Permite o usuário selecionar um arquivo. Use accept para filtrar tipos.

<input type="file" accept=".pdf,.jpg">
💡 Sempre use <label>! O label conecta o texto ao input. Quando o usuário clica no texto, o campo recebe o foco. Use o atributo for no label com o mesmo valor do id do input — ou envolva o input dentro do label.
02

Atributos importantes

Os atributos controlam o comportamento dos campos — validação, valor padrão, obrigatoriedade e muito mais. Tudo isso sem JavaScript.

Atributo O que faz Exemplo
name Nome do campo no envio do formulário. Obrigatório para o dado ser enviado. name="email"
id Identificador único na página. Usado para conectar com o <label for="...">. id="campo-email"
placeholder Texto de exemplo exibido quando o campo está vazio. Desaparece ao digitar. placeholder="Ex: Ana"
required Torna o campo obrigatório. O navegador bloqueia o envio se estiver vazio. required
value Define um valor padrão já preenchido no campo. value="Brasil"
disabled Desabilita o campo — o usuário não consegue interagir e o dado não é enviado. disabled
readonly Campo visível e selecionável mas não editável. O dado é enviado normalmente. readonly
min / max Define o valor mínimo e máximo para inputs numéricos e de data. min="1" max="100"
minlength / maxlength Define o mínimo e máximo de caracteres para campos de texto. minlength="8"
pattern Valida o campo com uma expressão regular (regex). Poderoso mas avançado. pattern="[0-9]{8}"
autocomplete Sugere valores salvos pelo navegador. Use off para desativar em senhas. autocomplete="off"
autofocus Foca automaticamente no campo quando a página carrega. autofocus
📄 HTML — exemplo com vários atributos
<form action="/cadastro" method="POST">

  <label for="nome">Nome completo</label>
  <input
    type="text"
    id="nome"
    name="nome"
    placeholder="Ex: João Silva"
    required
    minlength="3"
    autofocus
  />

  <label for="email">E-mail</label>
  <input
    type="email"
    id="email"
    name="email"
    placeholder="voce@email.com"
    required
    autocomplete="email"
  />

  <button type="submit">Enviar</button>

</form>
⚠️ action e method: o action define para onde os dados vão (URL do servidor). O method define como: GET coloca os dados na URL (visível), POST envia no corpo da requisição (seguro para dados sensíveis).
03

CSS para formulários

Inputs têm estilo padrão do navegador que costuma ser feio e diferente em cada sistema. O CSS abaixo mostra como controlar cada aspecto visual.

🖼️ Aparência base

Remove o estilo padrão e define borda, padding e fonte do zero.

input, select, textarea {
  width: 100%;
  padding: 0.65rem 0.9rem;
  border: 2px solid #ccc;
  border-radius: 8px;
  font-family: inherit;
  font-size: 1rem;
  background: #fff;
  outline: none;
}
🎯 Estado :focus

Estiliza o campo quando o usuário clica nele. Nunca remova o foco sem substituir — é acessibilidade!

input:focus,
textarea:focus {
  border-color: #2563eb;
  box-shadow: 0 0 0 3px
    rgba(37,99,235,0.2);
}
❌ Estado :invalid

CSS puro aplicado quando o campo falha na validação HTML (ex: email errado ou campo required vazio).

input:invalid:not(
  :placeholder-shown
) {
  border-color: #ef4444;
  background: #fff5f5;
}

O :not(:placeholder-shown) evita mostrar o erro antes do usuário digitar.

✅ Estado :valid

Aplica estilo quando o campo está preenchido corretamente. Ótimo para dar feedback positivo.

input:valid:not(
  :placeholder-shown
) {
  border-color: #22c55e;
  background: #f0fdf4;
}
🚫 Estado :disabled

Estiliza campos desabilitados visualmente. O cursor not-allowed reforça o feedback ao usuário.

input:disabled {
  background: #f1f5f9;
  color: #94a3b8;
  cursor: not-allowed;
  border-color: #e2e8f0;
}
🎨 Placeholder

Estiliza o texto de exemplo dentro do campo. É um pseudo-elemento separado.

input::placeholder {
  color: #94a3b8;
  font-style: italic;
  font-size: 0.9em;
}
🎚️ accent-color

Muda a cor de checkboxes, radios e range sliders com uma linha de CSS. Muito mais fácil que customizar do zero!

input[type="checkbox"],
input[type="radio"],
input[type="range"] {
  accent-color: #2563eb;
}
📐 Layout com Flexbox

Agrupe campos lado a lado com flexbox ou grid. Padrão para colocar campos como nome/sobrenome na mesma linha.

.form-linha {
  display: grid;
  grid-template-columns:
    1fr 1fr;
  gap: 1rem;
}
✅ Dica de acessibilidade: sempre use outline: none junto com um box-shadow ou border no :focus. Remover o outline sem substituir prejudica usuários que navegam pelo teclado.

Roteiro da aula

01
Contexto — por que forms existem? (5 min)

Mostre exemplos reais: login, cadastro, Google Forms, checkout de loja. Toda coleta de dados na web usa <form>.

02
Estrutura básica: form, label, input, button (10 min)

Escreva ao vivo o esqueleto com name, id e label conectado. Mostre o que acontece sem o atributo name no DevTools.

03
Tour pelos tipos de input (15 min)

Vá adicionando cada tipo na página e mostre o comportamento no navegador — especialmente date, range, radio e checkbox.

04
Atributos: required, placeholder, disabled... (10 min)

Adicione required em um campo e tente enviar o form vazio. Mostre a validação nativa do navegador funcionando.

05
Projeto ao vivo — Ficha de Cadastro (30 min)

Construa o formulário do projeto junto com os alunos. Comece sem CSS, depois aplique os estilos progressivamente. Mostre :focus e :invalid ao final.

06
CSS — estados e pseudo-classes (10 min)

Demonstre :focus, :hover, :disabled e :invalid no DevTools. Mostre como o box-shadow cria o efeito de "glow" no foco.

07
Desafio para casa 🏠

Criar um formulário de contato com: nome, email, assunto (select com 3 opções), mensagem (textarea) e botão. Estilizar com CSS próprio.
