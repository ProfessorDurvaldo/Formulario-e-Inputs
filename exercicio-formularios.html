01

Contexto do exercício

Você foi contratado para desenvolver a página de inscrição de um evento de tecnologia da sua cidade. O organizador precisa coletar algumas informações dos participantes antes do evento, e o formulário deve ser claro, funcional e com visual cuidadoso.

🎯 Objetivo Criar dois arquivos (index.html e style.css) que, juntos, formem um formulário de inscrição completo e estilizado. Não é necessário usar JavaScript — o foco é HTML e CSS.
02

Requisitos — HTML

Seu formulário deve conter todos os campos abaixo, com os atributos indicados corretamente configurados.

Campo Tipo / Elemento Requisitos obrigatórios
Nome completo input type="text" required, minlength="5", placeholder, autofocus, label associado com for/id
E-mail input type="email" required, placeholder, autocomplete="off", label
Data de nascimento input type="date" required, label
Cidade de origem input type="text" placeholder, label
Área de interesse <select> Pelo menos 4 opções (ex: Web, Mobile, IA, Redes), required, label, primeira option vazia como placeholder
Turno preferido input type="radio" 3 opções (Manhã / Tarde / Noite), mesmo name em todos, cada um com label e id únicos
Experiência prévia input type="checkbox" Pelo menos 3 opções (ex: HTML, JavaScript, Python), cada uma com label e id únicos
Horas de estudo/semana input type="range" min="1", max="40", label
Observações <textarea> placeholder, rows e cols definidos, label
Aceitar os termos input type="checkbox" required, label descritivo
Botões <button> Um type="submit" e um type="reset"
⚠️ Atenção Todo campo deve ter um <label> corretamente vinculado usando os atributos for e id. Labels soltos, sem vinculação, não serão considerados.
03

Requisitos — CSS

O CSS deve estar em um arquivo separado (style.css), vinculado ao HTML via <link>. Os estilos abaixo são obrigatórios:

  • Estilizar input, select e textarea com border, border-radius e padding consistentes
  • Aplicar efeito visual no :focus dos campos — o campo deve se destacar visivelmente quando o usuário clica nele (borda colorida ou box-shadow)
  • Estilizar o estado :disabled em algum campo — adicione um campo com disabled e aplique um estilo diferente (fundo cinza, cursor not-allowed, etc.)
  • Usar accent-color para dar cor aos checkbox, radio e range
  • Estilizar os botões de forma que se destaquem visualmente (cor de fundo, border-radius, cursor: pointer). Os dois botões podem ter estilos diferentes entre si
  • O formulário deve ter largura limitada e estar centralizado na página (uso de max-width + margin)
✅ Dica de ouro Use as pseudo-classes que vimos em aula: :focus, :disabled, ::placeholder. Elas são o coração do CSS para formulários!
04

Como será corrigido

Critério Descrição Impacto
Label vinculado Cada campo tem seu <label for="..."> correspondente ao id do input obrigatório
Radio com name Todos os radios de um mesmo grupo compartilham o mesmo atributo name obrigatório
CSS em arquivo externo Nenhum estilo pode estar inline ou dentro de <style> no HTML obrigatório
Campos obrigatórios Campos marcados com required na tabela devem realmente bloquear o envio se vazios obrigatório
Identação do código O HTML deve estar corretamente identado (use o Prettier do VS Code se quiser ajuda) obrigatório
Criatividade visual Formulário com identidade visual coerente e agradável pode render pontos extras a critério do professor ✨ bônus
05

Entrega

📁 O que entregar

  • index.html — o formulário com todos os campos solicitados
  • style.css — o arquivo de estilos separado, vinculado ao HTML
  • Os dois arquivos devem estar na mesma pasta, sem subpastas
  • Envie como .zip ou pelo canal indicado pelo professor
  • Nomeie o arquivo zip como seu nome + turma (ex: joao_silva_T1.zip)
